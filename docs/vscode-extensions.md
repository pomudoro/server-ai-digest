# Рекомендованные расширения VS Code

Расширения из `.vscode/extensions.json` — VS Code предлагает их установить при открытии проекта.

## astro-build.astro-vscode

Официальное расширение Astro. Подсветка синтаксиса `.astro` файлов, автодополнение компонентов и пропсов, TypeScript-диагностика, форматирование. Без него VS Code не понимает формат `.astro`.

## anthropic.claude-code

Claude Code — AI-ассистент от Anthropic, интегрированный в VS Code.

## esbenp.prettier-vscode

Prettier — автоформатирование кода при сохранении (отступы, кавычки, точки с запятой, переносы). Поддерживает JS, TS, HTML, CSS, JSON, Markdown и через плагины — `.astro`.

## dbaeumer.vscode-eslint

ESLint — линтер для JavaScript/TypeScript. Показывает ошибки и плохие практики прямо в редакторе, может автоисправлять при сохранении. Требует настройки ESLint в проекте (конфиг + пакет).

## yzhang.markdown-all-in-one

Удобная работа с Markdown: горячие клавиши (Ctrl+B — жирный, Ctrl+I — курсив), автодополнение списков, генерация оглавления, форматирование таблиц.

## davidanson.vscode-markdownlint

Линтер для Markdown. Проверяет структуру заголовков, пустые строки, trailing spaces, стили списков. Помогает держать `.md` файлы в чистоте.

## streetsidesoftware.code-spell-checker

Проверка орфографии в коде — подчёркивает опечатки в комментариях, строках, именах переменных и Markdown-тексте. По умолчанию только английский.

## streetsidesoftware.code-spell-checker-russian

Русский словарь для Code Spell Checker. Без него русский текст будет подчёркиваться как ошибка.

**Активация:** после установки обоих расширений нужно включить русский язык в настройках:

1. Открыть Settings (Ctrl+,)
2. Найти `cSpell.language`
3. Установить значение `en,ru`

Некоторые составные и технические слова (например, "автодополнение", "автоформатирование") могут всё равно подчёркиваться — их нет в стандартном словаре. Чтобы добавить слово, нажми Quick Fix (Ctrl+.) на подчёркнутом слове и выбери "Add to cSpell.json".

## usernamehw.errorlens

Показывает ошибки и предупреждения прямо в строке кода, а не только в панели Problems. Усиливает видимость сообщений от ESLint, markdownlint, spell-checker и других расширений.
