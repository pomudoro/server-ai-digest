# Batch Plan: 10 новостей для дайджеста

Дата: 2026-04-13

## Анализ приоритетов

Критерии из редполитики (`.claude/rules/article-style.md`):
- **high** — новые серверные продукты/платформы, GPU/ускорители, крупные анонсы вендоров, бенчмарки
- **medium** — значимые обновления охлаждения, сетей, памяти; практические гайды
- **low** — индустриальные стандарты, обзоры без конкретного продукта

Уже в блоге: Supermicro ARS-511GD-NB-LCC, Supermicro SYS-821GE-TNHR — учитываем при выборе порядка (разнообразие вендоров).

---

## 1. AMD Instinct MI350X — CDNA 4, 288 GB HBM3e

- **Приоритет:** high
- **Почему:** новый ускоритель, новая архитектура, прямой конкурент NVIDIA H200/B200 — ядро тематики блога
- **Ключевые факты:**
  - Архитектура CDNA 4 (следующее поколение после MI300X / CDNA 3)
  - 288 GB HBM3e — больше, чем у H200 (141 GB) и B200 (192 GB)
  - Позиционирование: инференс крупных моделей
  - Открытый ROCm-стек — важно для аудитории, привязанной к CUDA
  - Нужны бенчмарки: сравнение с H200/B200 по throughput и latency

## 2. NVIDIA DGX B200 / DGX SuperPOD

- **Приоритет:** high
- **Почему:** референсная платформа NVIDIA нового поколения, определяет стандарт рынка
- **Ключевые факты:**
  - 8x B200 GPU на узел (Blackwell)
  - NVLink Switch — пятое поколение NVLink, масштабирование на кластер
  - DGX SuperPOD — архитектура кластера (сколько узлов, какая сеть)
  - Сравнение с DGX H100: память, TDP, пропускная способность GPU-GPU
  - Форм-фактор, охлаждение, энергопотребление узла

## 3. Dell PowerEdge XE9680L — 4U liquid-cooled, 8 GPU

- **Приоритет:** high
- **Почему:** конкретный серверный продукт от крупного вендора, интересный форм-фактор
- **Ключевые факты:**
  - 4U вместо типичных 8U для HGX-платформ — как Dell упаковал 8 GPU
  - Liquid-cooled (direct-to-chip) — обязательное условие при такой плотности
  - Поддержка H200 и B200
  - Сравнение с 8U-решениями: плотность на стойку, стоимость инфраструктуры
  - Хорошо дополняет уже написанную статью про Supermicro SYS-821GE-TNHR (8U)

## 4. Intel Gaudi 3 — 128 GB HBM2e, встроенная RoCE

- **Приоритет:** high
- **Почему:** новый ускоритель от Intel, бюджетная альтернатива NVIDIA — интересно аудитории
- **Ключевые факты:**
  - 128 GB HBM2e (не HBM3e — отставание по памяти)
  - Встроенная сеть RoCE — не нужен отдельный ConnectX, снижение стоимости кластера
  - Позиционирование: инференс, а не обучение
  - Реальные бенчмарки: Llama 2/3 inference throughput vs H100/H200
  - Экосистема: Habana SynapseAI vs CUDA — барьер для миграции

## 5. NVIDIA Grace CPU Superchip (GH200/GB200 NVL)

- **Приоритет:** high
- **Почему:** ARM-процессор NVIDIA для серверов — меняет архитектуру GPU-серверов
- **Ключевые факты:**
  - Grace: 72 ядра Arm Neoverse V2, 480 GB LPDDR5X
  - Когерентная память CPU-GPU через NVLink-C2C — нет бутылочного горлышка PCIe
  - GH200: Grace + H200, GB200 NVL: Grace + 2x B200
  - Сценарии выигрыша: модели, не влезающие в GPU-память; рекомендательные системы; GNN
  - Сравнение с x86 + GPU: где ARM даёт реальное преимущество, а где нет

## 6. HBM4: переход с HBM3e

- **Приоритет:** medium
- **Почему:** ключевой компонент будущих ускорителей, но пока не продукт — перспектива
- **Ключевые факты:**
  - До 1.6 TB/s на стек (HBM3e — ~1.15 TB/s)
  - Samsung и SK Hynix — два ключевых производителя
  - Переход на новую архитектуру интерфейса (шире шина)
  - Ожидаемое появление: следующее поколение ускорителей (2026–2027)
  - Влияние на batch size, inference latency, стоимость за GB

## 7. Жидкостное охлаждение: direct-to-chip vs immersion

- **Приоритет:** medium
- **Почему:** критическая инфраструктурная тема при плотности 100+ кВт/стойка
- **Ключевые факты:**
  - Direct-to-chip (CoolIT, Vertiv): хладоноситель к холодным пластинам на GPU/CPU
  - Immersion (GRC, LiquidCool): весь сервер погружён в диэлектрик
  - Порог: ~40 кВт/стойка — воздух, 40–80 кВт — rear door, 80+ кВт — direct-to-chip/immersion
  - Капекс и опекс: переоборудование ДЦ, обслуживание, утечки
  - Кто из вендоров что поддерживает (Dell XE9680L — DTC, Supermicro — оба)

## 8. Сеть для AI-кластера: InfiniBand NDR400 vs Ultra Ethernet

- **Приоритет:** medium
- **Почему:** сеть — узкое место при масштабировании обучения, актуальный выбор
- **Ключевые факты:**
  - InfiniBand NDR400: 400 Gb/s, NVIDIA ConnectX-8, BlueField-4 DPU
  - Ultra Ethernet Consortium (UEC): открытый стандарт, AMD/Intel/Broadcom/Meta
  - RDMA over Ethernet vs native InfiniBand RDMA — разница в latency
  - Для кого InfiniBand (обучение, tight coupling), для кого Ethernet (инференс, cost-sensitive)
  - Текущие реалии: InfiniBand доминирует в обучении, Ethernet догоняет

## 9. On-prem AI-инфраструктура: минимальный кластер для файнтюнинга

- **Приоритет:** medium
- **Почему:** практический гайд, высокая ценность для аудитории (IT-закупщики, CTO)
- **Ключевые факты:**
  - Целевая задача: файнтюнинг моделей класса 70B (Llama 3 70B и подобные)
  - GPU: минимум 4–8x H100/H200 80–141 GB (один-два узла)
  - Сеть: InfiniBand NDR/HDR для multi-node, NVLink внутри узла
  - СХД: NVMe-массив или параллельная ФС (GPFS, Lustre, BeeGFS) для датасетов
  - Питание: 10–20 кВт на узел, нужны PDU и UPS соответствующей мощности
  - Охлаждение: при 2+ узлах с H100 — минимум rear door heat exchanger

## 10. OCP и стандарты AI-серверов

- **Приоритет:** low
- **Почему:** индустриальные стандарты, менее конкретно чем продукты, но полезно для контекста
- **Ключевые факты:**
  - OCP (Open Compute Project) — открытые спецификации серверного оборудования
  - ORv3 — стандарт питания для стоек (48V DC bus bar)
  - GPU-базплейн: стандартизация крепления и питания GPU-модулей
  - Кто шипает OCP-совместимое: Meta (заказчик), Wiwynn, QCT, Supermicro (частично)
  - Влияние: снижение vendor lock-in, но пока нишевая история vs проприетарные платформы
